# 电商类目分析模式

## 触发条件
- 用户提到生意参谋/SYCM/类目分析/市场分析/淘宝数据
- 或提供了SYCM页面URL/截图

## TAP时间锚点
- 每个Agent的task prompt注入当前UTC时间 + UTC+8（中国电商平台）
- SYCM数据标注采集日期（生意参谋数据以统计周期为准）
- web_search行业数据用freshness="pm"
- 1688价格数据标注采集日期（价格波动大）

## 外部依赖与降级方案

### 主路径：sycm-market-analysis skill
详见 sycm-market-analysis skill，核心流程：
1. Chrome Relay 或 managed browser (CDP) 连接SYCM
2. DOM抓取搜索排行9页(90条)
3. DOM抓取商品排行5页(50条)  
4. 关键词深钻10词×6维度(50-60条关联词)

**⚠️ 深钻关键陷阱：** search_analysis 是 SPA 页面，必须从 search_rank 点击"搜索分析"链接进入新 tab，然后在该 tab 内逐词 navigate。不能从空白页/search_rank 直接 navigate 到 search_analysis URL。详见 PLAYBOOK.md §1.7。

### 降级方案（sycm-market-analysis不可用时）

**触发条件：** SYCM登录态失效 / browser连接失败 / skill文件缺失

```
降级路线：

Level 1 — SYCM部分可用（搜索排行可采但深钻失败）
  → 只用搜索排行90条数据 + web_search补充关键词趋势
  → 深钻维度改用1688搜索词联想 + 淘宝下拉词
  → 标注"深钻数据为公开搜索补充，精度低于SYCM原始数据"

Level 2 — SYCM完全不可用
  → 数据源切换：
    a. 淘宝搜索前端页面（web_fetch抓取排序结果，取Top50商品）
    b. 1688关键词工具（web_search "1688 {关键词} 热搜"）
    c. 抖音/小红书热搜关联词（web_search平台热搜）
    d. 第三方公开报告（web_search "{类目} 市场分析报告 2026"）
  → 分析框架不变（QUANT/INTEL/DEMAND照常执行），但数据源标注[非SYCM]
  → VERIFIER必须标注数据来源差异，P6审查考虑数据降级影响
  → 最终报告顶部声明："本次分析基于公开数据，未使用SYCM原始数据，精度可能低于标准分析"

Level 3 — 所有电商数据源都不可用（极端情况）
  → 模式自动切换到C（竞品调研），用纯web_search做行业级分析
  → 放弃SKU级别的精细分析，只做类目级别的市场概览
  → 通知用户："SYCM不可用，已降级为行业概览模式"
```

## 固定团队（3个）

### QUANT-1 量化建模
- 四维评分：增速×0.20 + 人气×0.20 + 点击率×0.25 + 转化率×0.35
- K-Means聚类(k=3~8, 肘部法则)
- Monte Carlo模拟(1000次, UV/转化率均匀分布)
- 盈亏平衡+利润敏感度

### INTEL-1 竞品情报
- Top50品类结构+品牌集中度CR
- 流量效率矩阵(访客×买家四象限)
- 竞争壁垒评估(品牌/技术/白牌)
- Top5店铺深度拆解

### DEMAND-FORECAST 需求预测
- 修饰词TF-IDF+需求语义聚类
- 搜索路径漏斗还原
- 季节热度曲线
- 6个月现金流+Monte Carlo回本天数

## 动态团队（2-4个ANALYST）

### 赛道分组维度选择

先扫描90条关键词，按最显著特征分组：

| 类目特征 | 分组维度 | 示例赛道 |
|---------|---------|---------|
| 有消耗品+耐用品 | 复购频率 | 消耗品/耐用品/配件 |
| 有季节波动 | 季节性 | 常青品/季节品 |
| 价格跨度>10倍 | 价格带 | 引流款/利润款/形象款 |
| 多使用场景 | 功能场景 | 按场景分组 |
| 有人群差异 | 人群 | 男/女/专业/入门 |
| 时尚设计类 | 风格 | 按风格/材质分组 |

### 每个ANALYST的分析框架
1. 数据画像(增速/转化/点击/人气)
2. 竞品调研(淘宝TOP+1688批发价)
3. 利润空间计算(售价-成本-35%扣点)
4. 白牌机会评估(品牌占比/价格带空白)
5. 备货建议(首批/补货/季节窗口)
6. 关联销售链路(引流→利润→复购)
7. P0/P1/P2/P3优先级排序

## 分层架构（DNA Gene 1适配）
Layer 4  Knowledge Evolution — 类目认知积累/选品方法论结晶/赛道pattern沉淀
Layer 3  Cross-Validation — 多数据源交叉验证(SYCM+1688+生意参谋)/趋势vs销量矛盾识别
Layer 2  Domain Processing — 搜索排行分析/商品排行分析/价格段分析/关联词深钻/竞品拆解
Layer 1  Multi-Source Collection — SYCM DOM抓取/1688溯源/竞品采集/评论采集

## 领域红线（DNA Gene 5适配）
🔴 基础红线：
- SYCM账号登录态不得暴露在task prompt中
- 采集频率遵守平台限制，不触发反爬（每页≥2秒间隔）
- 竞品店铺数据应保护隐私，不在web_search中暴露店铺名
🟡 领域红线：
- 价格数据必须标注采集时间（价格波动大）
- 搜索排行数据TTL=7天（周维度更新）
- 商品排行数据TTL=1天（日维度更新）
- 转化率数据必须标注样本量（小样本不可信）
🟢 最佳实践：
- 图表中文字体用SimHei/Arial Unicode MS
- 价格用人民币¥标注
- 增速数据标注基期

## 部署梯度（DNA Gene 6适配）

## 领域协议（DNA Gene 2适配）
通用协议继承：MVP-P / SEP / DKR / WaC / RSP（全部适用）
领域特有协议：
- SYCM数据规范协议：DOM抓取优先于API / SPA tab处理规范 / 普惠版功能边界
- 赛道动态分组协议：不预设消耗品/季节品/配件，根据数据特征动态分组

## 维度覆盖检查（电商类目专用）
□ 搜索趋势：搜索排行TOP N词+增速+转化
□ 商品表现：交易量+交易增速+价格段分布
□ 竞品格局：头部品牌占比+白牌机会+差异化点
□ 供应链：1688源头价+毛利空间+MOQ
□ 季节性：月度趋势+节假日脉冲
□ 用户需求：关联词分析+搜索意图分类
□ 价格策略：价格段热力图+引流款vs利润款
□ 行动建议：SKU矩阵+定价建议+上架时机

## Prompt模板（DNA Gene 4适配）

### QUANT（量化建模师）
```
你是电商类目量化建模专家。任务：对{类目}进行数据驱动的量化分析。

数据源：读取 {workspace}/sycm-fetch/data/ 下的JSON文件
当前时间：{UTC_TIME} / {UTC8_TIME}

执行框架：
1. 四维评分模型：增速×0.20 + 人气×0.20 + 点击率×0.25 + 转化率×0.35
2. K-Means聚类(k=3~8, 肘部法则自动选k)
3. Monte Carlo模拟(1000次, UV/转化率均匀分布±20%)
4. 盈亏平衡计算 + 利润敏感度分析(成本/售价/转化各±10%)

输出要求：
- 所有数据标注采集时间和TTL
- 价格用¥标注，增速标注基期
- 置信度五级标注（⭐~⭐⭐⭐⭐⭐）
- 输出JSON + 摘要MD双格式

红线：
- 不在web_search中暴露竞品店铺名
- 转化率数据必须标注样本量
```

### DEMAND（需求预测师）
```
你是电商需求预测专家。任务：对{类目}进行需求侧深度分析和预测。

数据源：读取 {workspace}/sycm-fetch/data/ 下的JSON文件 + web_search补充
当前时间：{UTC_TIME} / {UTC8_TIME}

执行框架：
1. 修饰词TF-IDF提取 + 需求语义聚类（功能需求/场景需求/人群需求/价格需求）
2. 搜索路径漏斗还原：搜索词→点击→加购→转化，识别漏损环节
3. 季节热度曲线：月度趋势+节假日脉冲（618/双11/年货节）
4. 6个月现金流预测 + Monte Carlo回本天数(P50/P80/P95)

输出要求：
- 需求分类表（需求类型/代表词/搜索量/增速/竞争度）
- 季节性日历（月份/热度/备货节点/营销节点）
- 现金流模型（首批投入/月销预估/回本天数区间）
- 置信度五级标注

红线：
- 搜索排行数据TTL=7天，超期必须重新采集
- 价格数据标注采集时间
- web_search用freshness="pm"获取近期数据
```
