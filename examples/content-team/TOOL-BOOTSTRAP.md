## ⚠️ 全局输出铁律

### 铁律1: 零废话
- **禁止**：过渡句（"让我们来看看"/"值得注意的是"/"总的来说"）、感叹号煽情、重复总结
- **禁止**：AI味浓重的句式（"在当今XXX的时代"/"随着XXX的发展"/"不仅…更是…"）
- **每段必须有数据锚点**：没有数字/来源/案例支撑的段落→删掉
- **能用表格说清的禁止用文字**：竞品数据、关键词对比、平台规格→全部表格化
- **字数红线**：分析类Section最多300字（表格不计），超出→砍废话不砍数据

### 铁律2: 数据必须可验证
- 每条数据标注：**来源URL/工具 + 查询日期 + 原始query**
- **竞品内容数据必须标注获取方式**：web_search片段 / web_fetch实际页面 / xiaohongshu MCP / 手动推算
- 未标注来源→自动 LOW 置信度
- **禁止**：引用数据不标来源、编造互动数据、用过期数据冒充当前

### 铁律3: 数字用表格
❌ 错误：`这篇笔记点赞3.2万，收藏1.8万，评论2100，另一篇点赞1.5万...`
✅ 正确：
| 笔记标题 | 点赞 | 收藏 | 评论 | 发布日期 | 来源 |
|---------|------|------|------|---------|------|
| XXX | 3.2万 | 1.8万 | 2100 | 2026-02-20 | xiaohongshu MCP |

### 铁律4: 内容必须去AI味
- 创作的内容（标题/正文）严禁AI典型句式
- 检查清单：无"首先/其次/最后"三段式、无"不仅…更…"、无"值得一提"、无过度emoji堆砌
- 标题必须口语化、有钩子感、像真人写的
- 正文节奏：长短句交替、有个人视角、有具体细节

## 可用工具

### A. 内容/竞品数据采集

| 工具 | 调用方式 | 说明 |
|------|---------|------|
| **web_search**（主力） | `web_search query="关键词" count=10` | 通用搜索 |
| **web_search 定向** | `web_search query="关键词 site:xiaohongshu.com" count=10` | 定向平台搜索 |
| **web_fetch** | `web_fetch url="URL"` | 抓取页面内容 |

**内容研究标准流程（每个话题）：**
```
Round 1: web_search query="话题 小红书 爆款 笔记" count=10
Round 2: web_search query="话题 抖音 热门 视频" count=10
Round 3: web_search query="话题 公众号 10万+" count=10
Round 4: web_search query="话题 关键词 搜索趋势" count=10
Round 5: 对关键结果用 web_fetch 读取详情，提取内容结构
```

**竞品内容分析流程：**
```
Round 1: web_search query="竞品账号名 小红书 爆款" count=10
Round 2: web_fetch 抓取爆款笔记页面
Round 3: 提取: 标题结构、正文钩子、配图风格、标签策略、评论区高频词
Round 4: 统计: 互动数据（点赞/收藏/评论比例）
```

## 搜索规则

### 规则1: 多源交叉验证
每个关键数据点（互动数据、热度、受众画像）至少用2个不同来源验证。

### 规则2: 不相关时换措辞重试
- 第1次：原始关键词
- 第2次：同义词替换
- 第3次：缩短关键词
- 3次都不相关 → 切换工具

### 规则3: 来源标注
```
[来源: web_search, query="xxx", 返回N条, 相关M条]
[来源: web_fetch, url="xxx"]
[来源: xiaohongshu MCP, method=xxx]
```

### 规则4: 失败不能静默跳过
失败 → 记录错误 → 切换备选工具 → 在报告中标注

### 规则5: 数据时效性
内容行业数据变化极快。超过3个月的热度/互动数据标记为 `[⚠️ 数据可能过时]`。
