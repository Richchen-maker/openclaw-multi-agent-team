# 电商数据采集团队 — 启动指南

> 数据是弹药，选品团队不再自己找数据，从仓库取。

## 快速启动

### 模式A：指令采集
```
老板: "采集[类目名]近30天搜索热词"
老板: "采集[类目名]行业大盘数据"
老板: "采集[竞品店铺]近7天数据"
```

### 模式B：定时采集
```
老板: "设置每天自动采集[类目名]数据"
```

### 模式C：查看已有数据
```
老板: "仓库里有什么数据"
老板: "导出[类目名]最近的热词数据"
```

## 团队角色

| 角色 | 职责 | 类型 |
|------|------|------|
| DISPATCHER | 任务拆解、调度、质量门控 | Lead自身协议 |
| MAPPER | 摸底数据源、制定采集策略 | sub-agent |
| SPIDER | 执行采集（插件/浏览器/API/爬虫） | sub-agent |
| REFINER | 数据清洗、标准化、去重 | sub-agent |
| WAREHOUSE | 数据入库、索引、归档 | sub-agent |
| SENTINEL | 监控告警、反爬对策、登录态管理 | cron + sub-agent |

## 前置条件

1. Chrome浏览器已登录淘宝商家账号（生意参谋可访问）
2. SYCM Pro Extension已安装并启用
3. OpenClaw Browser Relay已连接

## 数据仓库位置

```
data-collection-team/warehouse/
├── INDEX.md        ← 数据资产总目录（先看这个）
├── raw/            ← 原始数据
├── cleaned/        ← 清洗后可用数据
└── archive/        ← 历史归档
```
