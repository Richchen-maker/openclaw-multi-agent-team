# ARC Team — 反风控对抗团队 启动指南

## 团队定位
第4团队。技术本质：Anti-Risk Control Evasion（反风控对抗）。
面向电商平台的攻防对抗研究与工程执行，覆盖数据采集突破、自动化交易对抗、安全渗透三条技术路径。

## 与现有团队的关系
```
patent-team (专利)     ← ARC产出的技术方案可作为专利挖掘输入
ecommerce-team (选品)  ← ARC为选品提供竞品数据突破能力
data-collection-team   ← ARC是data-collection的"重武器库"，
                         当常规采集被拦截时，ARC介入突破
```

## 快速启动

### 启动命令
对YOUR_AGENT_NAME说：
- **"启动ARC团队，目标：[平台名] [任务描述]"**
- 例："启动ARC团队，目标：淘宝x-sign签名逆向"
- 例："启动ARC团队，目标：Shopee反爬突破，采集全量SKU"
- 例："启动ARC团队，评估目标平台防御等级"

### 运行模式
| 模式 | 触发 | 说明 |
|------|------|------|
| Mode A: 逆向突破 | "逆向[平台][目标]" | 全链路：侦察→逆向→指纹→验证→部署 |
| Mode B: 防御评估 | "评估[平台]防御" | 只做侦察+分析，输出防御等级报告 |
| Mode C: 应急响应 | "采集被拦了" / "[平台]升级了" | 快速诊断+修复，恢复数据通道 |

### 红线
1. **纵队C（渗透/撞库）的任务不在AI团队中执行** — 纯黑产，仅作架构设计参考
2. 所有逆向成果按机密级管理，不进开源仓库
3. 代理池凭证、账号池数据不写入blackboard明文
